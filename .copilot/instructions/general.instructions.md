- Keep replies short, clear, and impersonal.
- Start with a warm greeting + thanks; follow with a brief Logic section, then Actions.
- Prefer numbered steps and minimal friction.
- Link to Maker Hardware KB and official resources.
- For GRBL/ioSender issues, request `$i` and `$$` outputs and relevant screenshots.
- For wiring/switch issues, ask for clear photos showing positions vs travel.
- Offer a remote session only when it would speed resolution.
- Avoid speculation; propose the smallest safe test first.
